# Práctica: Diseño de Interfaces con Tailwind

## Contexto

Este proyecto parte de una práctica anterior de JavaScript donde ya existe:

- Una tabla de alumnos
- Filtros funcionales
- Un formulario para añadir alumnos

La lógica JavaScript ya funciona y NO debe modificarse salvo que sea estrictamente necesario para integrar el diseño.

El objetivo ahora es únicamente aplicar diseño con Tailwind CSS.

---

## Requisitos obligatorios

La web debe cumplir exactamente lo siguiente:

### 1. Estructura de páginas

Debe tener 3 páginas independientes:

- index.html → Página de Inicio
- tabla.html → Página con la tabla y filtros
- formulario.html → Página con el formulario

---

### 2. Estilo con Tailwind

- Usar Tailwind mediante CDN
- No usar CSS personalizado salvo casos mínimos imprescindibles
- No usar frameworks adicionales

---

### 3. Modo claro / modo oscuro

Debe incluir:

- Un botón visible para alternar modo claro / oscuro
- El modo debe mantenerse entre páginas
- Usar localStorage para guardar la preferencia
- Aplicar correctamente clases `dark:` de Tailwind
- La clase `dark` debe aplicarse en `<html>`

---

### 4. Responsive

Debe ser completamente responsive:

- Mobile first
- Tabla usable en móvil (scroll horizontal si es necesario)
- Navegación adaptable
- Diseño correcto en móvil, tablet y escritorio

---

### 5. Diseño original

- Diseño limpio y moderno
- Uso correcto de espaciado
- Tarjetas, sombras suaves, bordes redondeados
- Botones estilizados
- Navbar coherente en todas las páginas

No debe ser minimalista sin estilo. Debe notarse que hay diseño.

---

## Restricciones importantes

- No rehacer la lógica JS
- No añadir funcionalidades nuevas
- No complicar el proyecto innecesariamente
- Hacer solo lo necesario para aprobar
- Código limpio y organizado
- Reutilizar componentes (navbar, botón dark mode)

---

## Objetivo real

Aprobar la práctica cumpliendo exactamente los requisitos sin añadir extras innecesarios.
